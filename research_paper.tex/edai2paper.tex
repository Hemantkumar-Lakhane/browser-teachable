\begin{abstract}
The technical barrier to entry in machine learning prevents domain experts from building custom classification models. We present ModelForge AI Studio, a fully browser-native, no-code platform for edge image classification. The system executes data ingestion, dataset editing, augmentation, transfer learning, evaluation, and model export entirely within the client browser using TensorFlow.js and WebGL. It eliminates server dependencies and ensures data privacy by keeping all images and weights on the local device. The platform features an integrated layer for Explainable AI, providing step-by-step visualization of the inference pipeline, occlusion heatmaps, and epoch replay. We demonstrate a portable deployment workflow that exports trained models directly into five industry-specific templates covering agriculture, insurance, manufacturing, waste management, and general classification. ModelForge differentiates itself from existing web-based tools by offering user-configurable hyperparameters, manual review for synthetic data augmentation, and comprehensive validation reporting.
\end{abstract}

\section{Introduction}

The barrier to entry in machine learning remains remarkably high for domain experts. Non-experts such as farmers, insurance assessors, factory engineers, and educators possess valuable domain knowledge but cannot build custom machine learning models. Traditional machine learning workflows require programming skills, specialized hardware accelerators, and complex software dependencies that fall outside the expertise of these professionals \cite{Carney2020}. 

Cloud-based machine learning platforms create a secondary problem regarding data privacy. Sensitive images from medical facilities, factory floors, or insurance claims cannot be safely routed through external corporate servers. Edge computing addresses this privacy concern by processing data locally, but existing edge tools still require software installation and technical environment setup \cite{Lian2021}.

We introduce ModelForge AI Studio to solve both the technical and privacy barriers simultaneously. The platform runs entirely in the modern web browser and requires no coding, no software installation, and no backend server. Users can upload images, label them, train a classifier, review what the model learned, and deploy it seamlessly within a single browser tab \cite{Smilkov2019}. We provide five industry deployment templates to demonstrate the immediate real-world applicability of these exported models.

The primary contributions of this paper are:
\begin{enumerate}
    \item A fully browser-native no-code machine learning pipeline from data ingestion to model deployment with zero server dependency.
    \item An augmentation system with manual per-variant review and acceptance to ensure data quality.
    \item An integrated Explainable AI layer including pipeline inspection, occlusion heatmaps with focus categorization, epoch replay with weight restoration, and embedding separability analysis.
    \item A portable model export mechanism and five industry-specific deployment templates covering agriculture, insurance, manufacturing, waste management, and general use.
\end{enumerate}

\section{Related Work}

\subsection{Browser-Based Machine Learning}
The execution of neural networks directly within web browsers has become viable due to advancements in WebGL and WebAssembly backends. The release of TensorFlow.js enabled JavaScript applications to perform hardware-accelerated tensor operations locally \cite{Smilkov2019}. Google's Teachable Machine popularized browser-based transfer learning by offering an approachable interface for novices to train basic classifiers \cite{Carney2020}. Recent projects like asanAI have expanded on this concept by introducing offline-first toolkits \cite{Koch2025}. 

\subsection{Edge Inference and Privacy}
Processing data at the edge fundamentally mitigates the privacy risks associated with cloud computing. By keeping raw data on the user's device, client-side inference circumvents the need for data transmission over external networks. Frameworks like WebFed have explored cross-platform federated learning within web browsers using local differential privacy techniques \cite{Lian2021}. 

\subsection{Automated Machine Learning Platforms}
Commercial Automated Machine Learning platforms aim to simplify the model creation process. Tools like Roboflow and LandingLens provide end-to-end environments for data annotation and model training \cite{Roboflow, LandingLens}. Cloud providers offer enterprise-grade solutions such as Vertex AI, Azure AutoML, and SageMaker Canvas \cite{VertexAI, AzureAutoML, SageMakerCanvas}. Platforms like DataRobot focus heavily on predictive modeling and robust deployment pipelines \cite{DataRobot}. 

\subsection{Research Gap}
Existing browser-based tools like Teachable Machine and asanAI lack user-controllable hyperparameter configuration, inline dataset editing, multi-stage Explainable AI, and end-to-end portable deployment with industry-specific templates. ModelForge addresses all of these limitations in a single integrated platform.

\subsection{Human-Verified Augmentation Pipeline}
The platform provides a Dataset Studio that utilizes an HTML5 canvas-based inline editor. Users can crop regions of interest and adjust brightness and contrast before embedding extraction occurs. The Auto-Augmentation Studio generates four synthetic variants for selected images: a horizontal flip, a 140\% brightness increase, a 60\% brightness decrease, and a 2px blur. These operations are applied via canvas context filters. Every synthetic variant requires human approval. Users must explicitly accept or discard each generated variant in the review grid before it enters the training set.

\subsection{Transfer Learning via MobileNet Embeddings}
The system utilizes the MobileNet v1 architecture as its feature extractor \cite{Howard2017}. The entire backbone remains frozen during training. The system extracts a 1024-dimensional embedding for each input image. Here, $\Phi$ denotes the frozen MobileNet feature-extraction function that maps an input image to its embedding vector. The extraction is defined as:
\begin{equation}
e = \Phi(x)
\end{equation}
where $x \in \mathbb{R}^{224 \times 224 \times 3}$ represents the input image and $e \in \mathbb{R}^{1024}$ represents the extracted feature vector. This transfer learning approach is highly suitable for browser-side training because it avoids calculating gradients across the millions of parameters in the large backbone \cite{Shao2015, Dewan2023}.

\subsection{Embedding-Guided Dataset Quality Assessment}
Before training begins, the system assesses dataset quality by calculating distances between class embeddings. The class centroid is computed as:
\begin{equation}
\mu_c = \frac{1}{|C|} \sum_{e \in C} e
\end{equation}
The mean embedding represents the mathematical center of a class. The system calculates intra-class dispersion to measure how varied the images within a single class are:
\begin{equation}
D = \frac{1}{|C|} \sum_{e \in C} ||e - \mu_c||^2
\end{equation}
The inter-class separability is calculated using cosine distance converted to a percentage difference between two classes:
\begin{equation}
S = \left(1 - \frac{\mu_A \cdot \mu_B}{||\mu_A|| ||\mu_B||}\right) \times 100
\end{equation}
The user interface renders color-coded warning bars if any class pairs score below a 15\% separation threshold.

\subsection{Trainable Classification Head}
The classification head consists of four specific layers appended to the embedding output. The first layer is a Dense layer with user-configured units (default 128) using ReLU activation. The second layer is a Dropout layer with a user-configured rate (default 0, maximum 0.8). The third layer is a Dense layer with units set to the maximum of 8 or half the first layer's units (default 64) using ReLU activation. The final layer is a Dense layer containing $K$ units corresponding to the number of classes, utilizing the softmax activation function:
\begin{equation}
P(y=k|e) = \frac{\exp(w_k^T e + b_k)}{\sum_{j=1}^K \exp(w_j^T e + b_j)}
\end{equation}
The model is optimized using the categorical cross-entropy loss function:
\begin{equation}
L = -\sum_{i} \sum_{k} y_{ik} \log(p_{ik})
\end{equation}
Users can configure all hyperparameters via the interface, including the optimizer choice (Adam, SGD, or RMSProp), learning rate (default 0.0005), epochs (default 25), and batch size (default 16).

\subsection{Stage-Wise Pipeline Introspection and Explainable AI (XAI)}
ModelForge integrates multiple tools to allow users to understand what the model learned \cite{Zeiler2014}. The Pipeline Inspector visualizes five distinct stages of inference: the raw webcam feed, the 224x224 downscaled image, the normalized tensor visualized as an RGB grid, the 1024D vector plotted as a line chart with peak activation highlighted, and a horizontal bar chart of class probabilities.

The Occlusion Heatmap uses a sliding gray patch of size 40 and stride 24 to obscure the image. The confidence drop per patch is normalized and mapped to a color scale. The system classifies the visual focus as highly focused (concentration below 8\%), moderately focused, or diffuse. 

Epoch Replay stores a snapshot containing the epoch number, loss, accuracy, and serialized weights at the end of every epoch. A scrubbing slider restores the exact weights via the API and runs a prediction on a saved test embedding. The interface displays contextual insight labels such as indicating whether the weights are still near random or converged. 

The evaluation module computes a full confusion matrix mapping actual versus predicted outcomes and renders it as a color-coded table. It also generates a classification report detailing Precision, Recall, F1-Score, and Support for each class.

\section{Implementation and Deployment}

\subsection{Browser Resource-Aware Design}
The system accelerates tensor operations using the WebGL backend provided by TensorFlow.js. Freezing the MobileNet backbone completely eliminates the need for backpropagation through its parameters. The codebase employs active tensor disposal techniques to prevent memory leaks during epoch iteration. Performance scales directly with the hardware capabilities of the host device. Image pixels are normalized from a range of [0, 255] to [-1.0, 1.0] by dividing by 127.5 and subtracting 1.0 to match the expected input distribution of the MobileNet model.

\subsection{Model Persistence}
ModelForge securely saves the active model weights directly into the browser's IndexedDB storage. The associated metadata is saved to local storage. This local persistence allows users to close the browser entirely and resume training or prediction later without data loss.

\subsection{Model Export and Portable Deployment}
Users can package their trained models into a downloadable ZIP file. This portable export contains five files: the topology structure, the binary weights, the metadata configuration, a portable inference page, and deployment instructions. The portable inference page acts as a self-contained runtime environment requiring only a browser. The entire exported package can be hosted on static platforms like GitHub Pages, Netlify, or Vercel simply by dragging and dropping the folder.

\subsection{Industry-Specific Deployment Templates}
ModelForge provides five pre-built deployment templates that consume the exported models. The agricultural scanner detects crop diseases for offline use in remote fields where internet access is unavailable. The claim scanner classifies vehicle damage for insurance routing while keeping claimant photos private. The quality assurance scanner detects manufacturing defects on assembly lines ensuring proprietary factory data never leaves the device. The waste scanner classifies garbage into recyclable, organic, and hazardous categories for real-time edge cameras in smart bins. The custom scanner serves as a generic template for any proprietary classification task.

\section{Limitations and Future Work}

The system relies entirely on the computational limits of the user's browser, meaning large datasets may cause memory crashes on low-end devices. The frozen MobileNet backbone is optimized for general imagery and struggles to differentiate fine-grained domain-specific textures. The current pipeline supports only image classification and does not support object detection or segmentation. Execution speeds vary significantly depending on browser background resource allocation and available WebGL support.

Future work will focus on expanding the platform to support object detection and image segmentation pipelines. We plan to conduct formal user studies with non-expert participants across the five specific deployment domains to measure usability. We intend to support additional backbone architectures including EfficientNet and MobileNet v2. We also aim to introduce federated learning capabilities to enable collaborative model training across multiple browser clients.

\section{Conclusion}

ModelForge AI Studio is a decentralized edge computing platform that democratizes image classification for domain experts. The system introduces an advanced Explainable AI layer, augmentation with manual per-variant review, and a fully configurable training architecture. The platform exports portable models into five industry-specific templates with zero server dependency. Experimental evaluation demonstrated that the platform achieved an F1-score of [RESULT: X]. Future iterations will explore object detection and federated learning integration.
